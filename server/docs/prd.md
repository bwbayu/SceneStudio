# PRD: Cine-Agent (AI-Powered Interactive Story Platform)

**Project Vision:** To create a "Roblox for Interactive Films" platform where AI acts as a complete film crew to transform raw scripts into professional-quality FMV (Full Motion Video) games.

---

## 1. System Summary

Cine-Agent utilizes a **Multi-Agent Orchestration** architecture. The system processes user raw scripts through a reasoning loop to ensure detail completeness, then produces 5-7 branching scenes. Each scene consists of 3 video segments (~8 seconds per segment) that are seamlessly connected using the **Extend** and **Native Audio** features of **Veo 3.1**.

---

## 2. Multi-Agent Roles (The AI Studio)

The system relies on specific task distribution among agents to accurately populate the Pydantic data structure.

* **Director Agent (The Orchestrator):** The central agent performing script analysis. It is responsible for the **Dynamic Clarification** flow and performing final validation of the entire data structure before it is sent to the rendering stage.
* **Screenwriter Agent:** Tasked with structuring the plot, determining the number of scenes (5-7), and building the branching story logic.
* **Casting & Production Designer:** Generates in-depth visual descriptions for every actor and setting (theme) to maintain visual consistency.
* **Segment Engineer:** A technical specialist who breaks down each scene into 3 separate video segments with camera movement instructions and specific actions optimized for Veo 3.1 prompts.

---

## 3. Key Feature: Dynamic AI Clarification

Unlike static systems, the clarification process is now fully driven by LLM intelligence:

* **AI-Generated Questions:** The Director Agent analyzes the raw script. If ambiguities are found (e.g., "Nina feels sad in her room," the AI might ask "What is the atmosphere of Nina's room?"), the agent dynamically generates questions.
* **Dynamic Choices:** The agent also generates relevant and creative answer choices (A, B, C) to help the user detail the story quickly.
* **Tool Integration:** Uses the `AskUserQuestion` function which triggers a UI modal to receive direct input from the user.

## 4. Data Structure

### A. Main Model: `StoryBoard`

| Field | Type | Description |
| --- | --- | --- |
| `story_id` | `str` | Unique UUID identifying the story project. |
| `title` | `str` | Title of the film or story created. |
| `actors` | `list[Actor]` | List of all unique characters/actors involved in the story. |
| `themes` | `list[Theme]` | List of unique locations or environmental settings used. |
| `scenes` | `list[Scene]` | Collection of scenes (5-7 scenes) forming the story tree. |

### B. Character Model: `Actor`

| Field | Type | Description |
| --- | --- | --- |
| `actor_id` | `str` | Unique ID for actor reference (e.g., `actor_01`). |
| `name` | `str` | Name of the character in the story. |
| `physical_description` | `str` | Permanent physical description (race, hair, facial features) for AI prompts. |
| `outfit_description` | `str` | Description of the clothing worn to remain consistent in every scene. |
| `anchor_image_url` | `str?` | URL of the reference face image generated by Gemini Flash Image. |

### C. Location Model: `Theme`

| Field | Type | Description |
| --- | --- | --- |
| `theme_id` | `str` | Unique ID for location reference (e.g., `theme_cafe`). |
| `location_name` | `str` | Name of the location (e.g., "School Canteen"). |
| `atmosphere` | `str` | Visual mood (e.g., *Cyberpunk, Warm, Gloomy*). |
| `lighting` | `str` | Light settings (e.g., *Golden hour, Harsh neon*). |
| `reference_image_url` | `str?` | URL of the background reference image for consistency. |

### D. Scene Model: `Scene`

| Field | Type | Description |
| --- | --- | --- |
| `scene_id` | `str` | Unique ID for the scene (e.g., `scene_01`). |
| `title` | `str` | Name or short title of the scene. |
| `segments` | `list[Segment]` | List of 3 video segments (Intro, Mid, End) forming the scene. |
| `choices` | `list[Choice]?` | List of interactive choices appearing at the end of the scene. |
| `is_ending` | `bool` | Marker if this scene is the end of a story branch. |

### E. Video Segment Model: `Segment`

| Field | Type | Description |
| --- | --- | --- |
| `segment_index` | `int` | Segment order within a scene (1, 2, or 3). |
| `visual_prompt` | `str` | Primary technical prompt sent to the Veo 3.1 API. |
| `camera_movement` | `str` | Camera movement instructions (e.g., *Pan right, Close-up*). |
| `action_description` | `str` | Specific action performed by the actor in this segment. |
| `video_url` | `str?` | URL of the final rendered Veo video file (includes native audio). |

### F. Choice Model: `Choice`

| Field | Type | Description |
| --- | --- | --- |
| `text` | `str` | Text displayed on the choice button on screen. |
| `target_scene_id` | `str` | Target scene ID if this choice is selected by the user. |

---

## 5. Technical Production Pipeline

1. **Input:** User enters a raw script idea.
2. **AI Clarification Loop:** Director Agent analyzes the script. If data is missing for `Actor` or `Theme`, the agent calls `AskUserQuestion` with dynamically generated questions and choices.
3. **Data Generation:** Once the script is complete, specialist agents work in parallel to populate the data structure tables above.
4. **Visual Anchoring:** Gemini Flash Image creates actor reference images as consistency "anchors."
5. **Sequential Video Generation (Veo 3.1):**
* **Segment 1:** Generate initial video.
* **Segment 2:** Generated using the **Extend** feature from Segment 1.
* **Segment 3:** Generated using the **Extend** feature from Segment 2 (ending with an interaction trigger action).


6. **Game Assembly:** Metadata and video assets are assembled into an interactive playable application.
